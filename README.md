# Timeline Viewer

Локальное веб-приложение для просмотра событий на временной шкале и связанных медиафайлов.

## Назначение

- Загрузка папки с событиями.
- Отображение событий на таймлайне.
- Просмотр фото и видео, привязанных к выбранному событию.

## Формат данных

- Каждое событие — отдельная папка.
- Имя папки содержит дату или диапазон дат в квадратных скобках:
  - `[YYYY-MM-DD] Название`
  - `[YYYY-MM-DD/YYYY-MM-DD] Название`
- Дополнительные теги задаются квадратными скобками до блока даты:
  - `[tag1][tag2][YYYY-MM-DD] Название`

## Возможности

- Выбор папки через диалог или drag-and-drop.
- Фильтрация событий по тегам.
- Масштабирование таймлайна колесом мыши.
- Панель миниатюр с открытием в полноэкранном просмотрщике.
- Ручное изменение высоты блоков таймлайна и миниатюр через разделитель.

## Запуск

Статический клиент без сборки. Достаточно открыть `index.html` в браузере.

## Скрипты

### `create_tree_report.sh`

Создает текстовый отчет со структурой проекта:

- Имя файла отчета: `tree_YYYY-MM-DD.txt`.
- В первой строке пишет количество директорий и файлов.
- Если установлен `tree`, использует его; иначе строит дерево через `find`.

Запуск:

```bash
./create_tree_report.sh
```

### `rename_files_menu.sh`

Интерактивная утилита для массового переименования медиафайлов (фото и видео) в текущей папке и подпапках.
При запуске создает лог-файл `media_renamer_YYYYMMDD_HHMMSS.log`.

Доступные режимы:

- `1` — переименование по EXIF:
  обрабатывает фото с EXIF-датой и приводит имя к формату `YYYY-MM-DD_HH:MM:SS.ext` (или `..._123.ext` при наличии SubSec).
- `2` — переименование по шаблонам имен:
  распознает распространенные имена скриншотов/камер (например `IMG_`, `VID_`, `Screenshot`, `photo`, `video`) и приводит к единому формату.
- `3` — переименование по метаданным файла:
  использует наиболее раннюю дату из метаданных (создание/изменение), имя формируется как `ns_YYYY-MM-DD_HH:MM:SS.ext`.

Во всех режимах используется безопасное переименование с обработкой конфликтов имен.

Запуск:

```bash
./rename_files_menu.sh
```

Зависимости:

- Для режима `1` требуется `exiftool`.
